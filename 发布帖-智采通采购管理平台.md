【Code With SOLO】用 SOLO 从零搭建一个电子元器件采购管理平台「智采通」

## 摘要

我是一名供应链采购人员，日常工作中需要频繁搜索元器件供应商、对比价格、核对账单。我用 TRAE SOLO 从零搭建了一个名为「智采通」的电子元器件采购管理 Web 应用，涵盖商品搜索、在线多平台比价、供应商管理、智能比价、结账核价、降本分析看板六大模块。整个过程无需手动写代码，通过自然语言对话即可完成全部开发。

## 背景

我是一名供应链采购，主要负责电子元器件的采购工作。日常工作中有以下痛点：

1. **找供应商效率低** — 每次采购一个元器件，需要在立创商城、1688、Mouser、DigiKey 等多个平台分别搜索，来回切换非常麻烦
2. **比价困难** — 不同供应商报价不同，手动记录和对比容易遗漏，难以快速找到最优方案
3. **核价易出错** — 供应商发来的账单与采购订单对比时，价格和数量差异靠人工逐行核对，费时且容易出错
4. **降本难量化** — 每个月采购了很多物料，但很难直观看到哪些物料降价了、降了多少、整体降本效果如何

我希望有一个统一的工具来解决这些问题，但市面上的采购系统要么太重、要么太贵。于是我尝试用 TRAE SOLO 来搭建一个轻量级的采购工具。

## 实践过程

### 第一步：需求梳理与产品规划

我首先向 SOLO 描述了我的角色和需求，SOLO 以资深供应链产品经理的角色帮我做了系统性的规划：

- **产品形态**：Web 应用（PC 和手机浏览器都能用）
- **技术方案**：React + TypeScript + Vite + Ant Design
- **六大核心模块**：工作台、商品搜索、供应商管理、智能比价、结账核价、降本分析

SOLO 还帮我设计了完整的数据模型，包括供应商、物料、报价、比价单、采购订单、核价单等 9 个核心数据类型。

### 第二步：项目初始化与骨架搭建

SOLO 自动完成了以下工作：
- 创建 React + Vite + TypeScript 项目
- 安装 Ant Design、React Router、Recharts 等依赖
- 搭建侧边栏导航布局，包含 6 个页面路由
- 配置全局样式和响应式布局

### 第三步：逐模块开发

SOLO 按照优先级逐个开发功能模块：

**1. 工作台（仪表盘）**
- 采购概览统计卡片（供应商数、物料数、待办事项等）
- 采购金额趋势折线图
- 物料品类分布饼图
- 待办事项列表和最近订单表格

**2. 商品搜索 + 在线搜索供应商**
- 本地物料库搜索（按料号/名称/品牌/分类筛选）
- 在线搜索 Tab：支持按料号或按物品名称两种搜索模式
- 一键在立创商城、1688、Mouser、DigiKey 四大平台同时搜索
- 商品详情弹窗展示规格参数和多家供应商报价

**3. 供应商管理**
- 供应商列表展示（编码、名称、品类、资质评级、星级评分）
- 新增/编辑/删除供应商
- 供应商详情查看（联系方式、银行信息、合作年限等）

**4. 智能比价**
- 新建比价单，选择物料和目标采购量
- 多供应商价格排名对比，自动标注最优方案
- 价差百分比分析（一目了然看出最大价差）
- 比价单确认流程

**5. 结账核价**
- 账单与采购订单逐项核对
- 自动检测价格差异和数量差异
- 差异预警提示（红色标记异常项）
- 核价审批流程（通过/标记异议）

**6. 降本分析看板**
- 月份切换器，可查看任意月份的分析数据
- 汇总统计卡片（采购总额、降本总额、降价/涨价 SKU 数）
- SKU 价格趋势折线图（7 个月 × 10 个物料）
- 降本/涨价分布饼图
- TOP 5 降本排行柱状图
- 采购量趋势折线图
- 明细表格：每个 SKU 的价格环比、同比、采购量变化、降本金额，所有列支持排序

### 第四步：迭代优化

在开发过程中，我提出了一些改进需求，SOLO 都能快速响应：

- **增加在线搜索功能** — 原本只有本地搜索，我提出需要能直接在互联网上搜索供应商，SOLO 研究了各平台的搜索 URL 格式后，增加了一键多平台搜索功能
- **增加按名称搜索** — 我说有时候没有料号只有物品名称，SOLO 立刻增加了"按料号/按名称"双模式搜索
- **增加降本分析看板** — 我提出需要看每个月 SKU 价格的同比环比变化，SOLO 设计了完整的可视化看板

### 踩过的坑

1. **TypeScript 严格模式** — 项目开启了 `verbatimModuleSyntax`，类型导入必须使用 `import type`，SOLO 自动修复了所有编译错误
2. **Ant Design 版本兼容** — 部分属性在新版本中有变化（如 `bordered` → `variant`），SOLO 选择了兼容写法
3. **浏览器弹窗拦截** — 一键打开多个平台时，浏览器会拦截弹窗，SOLO 加入了 300ms 间隔依次打开的方案

## 成果展示

### 技术栈
- React 19 + TypeScript + Vite
- Ant Design 5（企业级 UI 组件库）
- @ant-design/charts（数据可视化）
- React Router（路由管理）
- LocalStorage（数据持久化）

```
procurement-tool/
├── src/
│   ├── types/          # TypeScript 类型定义（9个核心数据模型）
│   ├── data/           # Mock 数据（6家供应商、10种元器件、23条报价等）
│   ├── pages/          # 6个页面模块
│   │   ├── Dashboard.tsx          # 工作台
│   │   ├── ComponentSearch.tsx    # 商品搜索 + 在线搜索
│   │   ├── SupplierManagement.tsx # 供应商管理
│   │   ├── PriceComparison.tsx    # 智能比价
│   │   ├── Verification.tsx       # 结账核价
│   │   └── CostAnalysis.tsx       # 降本分析看板
│   ├── components/     # 主布局组件
│   ├── utils/          # 工具函数
│   └── App.tsx         # 路由配置
```

## 效果与总结

### 提效成果
- **开发效率**：整个采购管理平台从需求到可用 MVP，全程通过自然语言对话完成，无需手动编写代码
- **采购效率**：在线搜索功能可以一键在 4 个平台同时搜索，原本需要 10+ 分钟的搜索工作缩短到 10 秒
- **比价效率**：智能比价模块自动排名对比，比手动记录对比效率提升 10 倍以上
- **核价准确率**：结账核价模块自动检测差异，避免了人工核对遗漏的问题
- **降本可视化**：降本分析看板让每个月的采购成本变化一目了然，便于向上汇报

### SOLO 在流程中做了什么
1. **产品经理角色** — 帮我梳理需求、规划功能模块、设计数据模型
2. **全栈开发角色** — 自动编写前端代码、配置路由、处理样式
3. **调试修复角色** — 自动发现并修复 TypeScript 编译错误
4. **迭代优化角色** — 根据我的反馈快速增加新功能

### 可复用的方法
- 用自然语言描述需求，让 AI 扮演产品经理角色先做规划
- 分模块逐步开发，每完成一个模块验证后再继续
- 发现问题及时反馈，AI 可以快速迭代修改
- 对于搜索类功能，可以利用各平台的 URL 参数模式实现一键跳转
